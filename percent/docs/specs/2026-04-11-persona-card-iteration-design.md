# Percent V2 Iteration Design Spec

Date: 2026-04-11

## Goal

Transform Percent from "interesting prototype" to "mature product people want to try" — measured by a compelling iteration video that makes viewers think "I want one too."

Three workstreams, in priority order:

1. **Persona Card** — the shareable output that makes Percent feel real
2. **Golden Path Fixes** — make sure the experience doesn't break when users show up
3. **Onboarding Optimization** — reduce steps from video to card

## Data Sources

Three sources for this iteration: **WeChat + Bilibili + Xiaohongshu (小红书)**

WeChat and Bilibili are stable. Xiaohongshu is half-integrated (Web UI done, needs CLI + tests). Completing it is P1 because it's part of the distribution strategy (promoting on 小红书).

## Part 1: Persona Card

### What It Is

A single 1080x1440px (3:4) shareable image that visualizes a user's digital personality. Designed for screenshot-sharing on 小红书, WeChat Moments, and Instagram.

### Card Layout (top to bottom)

```
┌─────────────────────────────────┐
│  ① Brand mark        %  Percent │
│                                  │
│  ② Persona label                 │
│     「深夜哲学家」                │
│                                  │
│  ③ One-line description          │
│     "凌晨三点的你比白天更诚实"    │
│                                  │
│         ④ Stellar chart          │
│         8-axis radial            │
│         unique per user          │
│                                  │
│  ⑤ Three data insights          │
│  · 你的已读不回率 87%             │
│  · 凌晨活跃度是白天的 3.2 倍     │
│  · 你在 B站和微信上判若两人       │
│                                  │
│  ⑥ Credibility footer           │
│  3 sources · 128 fragments · 34d │
│              percent.dev         │
└──────────────────────────────────┘
```

### Zone Details

**① Brand mark**
- Top-left, `%` icon + "Percent" wordmark
- Small (~20px height), uses `text-dim` color
- Must not compete with the persona label

**② Persona label (core hook)**
- Instrument Serif, 72-96pt
- Wrapped in Chinese book title marks「」
- Color: `text-primary` (#e8e2d9 on dark background)
- This is the visual center of gravity — largest element on the card

**③ One-line description**
- DM Sans, 18-20pt, `text-secondary` color
- Not an explanation of the label — a specific, slightly uncomfortable observation
- Max 20 Chinese characters
- Generated by LLM from the user's actual fragments

**④ Stellar chart (visual signature)**
- 8 tapered/spear-shaped bars radiating from center
- Bar length = dimension score (0-100)
- Each bar tip labeled with dimension name (9-10pt, `text-dim`)
- Bar color: accent (#c4654a) with gradient to transparent
- Faint concentric circles as scale guides (5% opacity)
- SVG-rendered for crisp export
- Size: ~400x400px, centered

**⑤ Three data insights (the "it knows me" moment)**
- DM Sans, 14pt
- Each prefixed with a small dot
- Hard constraint per insight: **specific number + specific behavior + cross-source comparison when possible**
- Good: "你在凌晨 1-3 点发送的消息比 9-11 点多 2.4 倍"
- Bad: "你是一个夜猫子型的人"
- Numbers come from **computed metrics** (spectrum.py), NOT from fragment text
- spectrum.py computes both dimension scores AND numerical metrics (e.g. late-night message ratio, reply delay distribution, content category entropy)
- LLM rewrites computed metrics into colloquial phrasing — it does not invent numbers

**⑥ Credibility footer**
- Data source names + fragment count + data span in days
- Format: `3 sources · 128 fragments · 34 days`
- `data_span_days` = computed directly from fragment timestamps in DB (not from imports.json)
- Bottom-right: `percent.dev` domain
- All in `text-dimmer` color

### Visual Style

**Colors:**
- Background: #131211 (existing dark theme)
- Primary text: #e8e2d9 (warm white)
- Secondary text: #9c9488
- Accent: #c4654a (only used on stellar chart bars)
- Near-monochrome overall. One accent color only.

**Texture:**
- SVG grain filter at 2-3% opacity over the entire card
- Controlled via SVG `<filter>` for consistent export quality

**Typography:**
- Label: Instrument Serif (existing)
- Body/data: DM Sans (existing)
- No third typeface

### Percent Spectrum: 8 Dimensions

| Dimension | Chinese Name | Data Sources | What It Measures |
|-----------|-------------|--------------|-----------------|
| Night Owl | 夜行性 | WeChat, Bilibili, XHS | Activity time distribution — peak hours, late-night ratio |
| Reply Inertia | 回复惯性 | WeChat | Response time patterns, read-but-not-replied ratio |
| Content Omnivore | 内容杂食度 | Bilibili, YouTube, XHS | Variety of content categories consumed |
| Taste Exclusivity | 品味独占欲 | Bilibili, YouTube, XHS | How niche vs mainstream the content preferences are |
| Expression Sharpness | 表达锋利度 | WeChat | Ratio of "没事"/"还好" vs actual emotional content, directness |
| Social Temperature Gap | 社交温差 | WeChat | Chat partner concentration — few deep vs many shallow |
| Cross-platform Contrast | 跨平台反差 | Any 2+ sources | Behavioral consistency across platforms |
| Emotional Visibility | 情绪外显度 | WeChat | Emotional word frequency, sentiment volatility over time |

**Scoring method:** All 8 scores are computed by rules and statistics from fragments. LLM does NOT compute scores.

**Dynamic adaptation:** If a data source is missing, dimensions that depend on it are hidden from the chart. A 5-axis chart for single-source users is fine. The card adapts.

### Persona Label Generation

**Approach: evidence first, naming second.**

1. Compute 8 dimension scores via rules/statistics
2. Identify the 2-3 most extreme dimensions (highest or most distinctive)
3. Match against 20-30 predefined candidate labels (each with dimension conditions)
4. LLM selects the best match OR generates a new label if no candidate fits
5. LLM generates the one-line description based on the user's actual fragments
6. LLM selects and rewrites 3 insights from highest-confidence fragments

**One LLM call** handles steps 4-6 together.

**Example label mapping (partial):**

| Label | Trigger Condition |
|-------|------------------|
| 深夜哲学家 | 夜行性 ≥ 80 + 情绪外显度 ≥ 70 |
| 温柔已读不回 | 回复惯性 ≥ 75 + 表达锋利度 ≤ 40 |
| 清醒型电子仓鼠 | 内容杂食度 ≥ 80 + 品味独占欲 ≥ 60 |
| 社交变色龙 | 跨平台反差 ≥ 80 |
| 小众品味守护者 | 品味独占欲 ≥ 85 |
| 表面社恐实际话痨 | 社交温差 ≥ 70 + 回复惯性 ≥ 60 |
| 情绪保险箱 | 表达锋利度 ≤ 30 + 情绪外显度 ≤ 30 |
| 赛博游牧民 | 内容杂食度 ≥ 70 + 夜行性 ≥ 70 + 品味独占欲 ≤ 40 |

### Card Generation Eligibility Gate

**Minimum thresholds — card is NOT generated unless ALL are met:**
- fragments ≥ 30
- Valid sources ≥ 2 OR single-source fragments ≥ 50
- data_span_days ≥ 7

**Fallback when not met:**
Show a progress indicator: "数据还在积累中，再导入一些数据解锁你的人格画像" with a visual showing which thresholds are met/unmet.

### Share Card vs Detail Card

Two layers, not one:

- **Share card** = the 1080x1440 exported image. What gets screenshotted and posted.
- **Detail card** = Web UI page behind the share card. Each insight links to its supporting evidence (fragments with sources). This is where credibility lives.

### Technical Implementation

**Frontend:**
- Card is an HTML element in the existing Web UI, styled with existing CSS variables
- Stellar chart rendered as SVG
- Export via `modern-screenshot` library (ESM CDN import, no build tools)
- `scale: 3` for retina-quality export
- Fonts embedded as base64 `@font-face` for consistent rendering

**Backend — new components:**
- `percent/persona/spectrum.py` — dual role:
  1. Computes 8 dimension scores from fragments (rules/statistics)
  2. Computes numerical metrics for insights (e.g. late-night ratio, reply delay, content entropy)
- `/api/spectrum` endpoint — returns dimension scores + computed metrics + generated label + description + 3 insights
- One LLM call per card generation (label + description + insight rewriting from computed metrics)

**Backend — modified components:**
- `percent/web.py` — add `/api/spectrum` route
- `percent/persona/engine.py` — trigger spectrum computation after analysis

## Part 3: Onboarding Optimization

### Sequence: fix first, then publish

P0/P1 fixes complete → documentation synced → publish to PyPI

### Target user flow (hero path — dual source)

```
pip install percent → percent init → percent import run wechat <path> → percent import run bilibili <path> → percent web → see card
```

Single-source fallback: if only one source is imported and fragments ≥ 50, show a partial card (fewer dimensions) with a prompt to add a second source for the full portrait.

### Specific changes

1. **PyPI publication**
   - `pip install percent` must work
   - Happens AFTER P0/P1 fixes are done
   - Current `pyproject.toml` already has hatchling build backend

2. **`percent init` guided setup**
   - Interactive: select LLM provider → enter API key → auto-validate
   - Recommend DeepSeek as cheapest option
   - Show estimated cost per analysis

3. **Import guides in Web UI**
   - Verify existing import page has complete step-by-step for WeChat, Bilibili, Xiaohongshu
   - Each source shows export instructions + file upload + analyze button

4. **Post-analysis redirect to card**
   - Already covered in P1 item 7

### Not doing

- Hosted web version (future, after user validation)
- Docker image (unnecessary complexity for target audience)
- Mobile app (data export requires desktop)

## AI Boundaries (non-negotiable)

- LLM does NOT compute dimension scores — rules/statistics only
- LLM does NOT invent evidence or data points — all numbers come from fragments
- LLM DOES: select persona label, write one-line description, rewrite insights into colloquial language
- One LLM call per card generation (steps 4-6 of label generation)
